INVESTIGACION

URL (Uniform Resource Locator)

Es una direccion en la que se encuentra algun archivo dentro del internet.[10]

MIME (Multi-Purpose Internet Mail Extensions)

Es una extension que permite enviar diferentes tipos de archivo como audio, video, imagenes, aplicaciones, programas.

HTTP(Hypertext Transfer Protocol)

Metodos : Sirviven para indicar una accion que se quiere hacer para un recurso en especifico.

GET: Recuperar datos, solicita una representacion de un recurso
HEAD : Solicita una respuesta igual a la de GET, sin el cuerpo de la respuesta
POST : Enviar una entidad a un recurso, cambio en el estado del servidor
PUT : Remplazar todas las representaciones del recurso destino con una nueva peticion
DELETE : Borrar el recurso
CONNECT : Hacer un tunel entre el servidor y el recurso
OPTIONS : Identificar las opciones de comunicacion para el recurso destino
TRACE : Prueba de bucle a traves de la ruta al recurso destino.
PATCH : Aplicar modificadores a un recurso[6]

Errores

4XX : Por el navegador
5XX : Por el servidor

XML(Extensible Markup Language)

Es un meta lenguaje el cual fue desarrollado por el Word Wide Web Consortiu. XML es una adaptacion del SGML el cual permite una mejor organizacion y etiquetado de documentos. Es decir, el XML no es un lenguaje, es un sistema que permite definir lenguajes.[2]

XHTML

ES un lenguaje parecido a HTML, sin embargo es mas aconsejable para la modelacion de paginas web. En otras palabras es una version mas pura y estetica de HTML[2]

HTML5

Es la ultima version de HTML, con nuevos elementos y comportamientos, ademas de que cuenta con una mejor tecnologia que permite a las paginas y aplicaciones web contar con una mayor diversidad y alcance.

JS (Java Script)

Es un lenguaje orientado a objetos que cuenta con funciomnes de primera clase, mejor conocido como lenguaje de script para paginas web. Tambien es usado en entornos sin navegador como "Apache". Este lenguaje soporta estilos de programacion funcional orientada a objetos.[1]

php (Hypertext Preprocessor)

Lenguaje utilizado para el desarrollo web, este puede ser incrustado en HTML.[3]

jsp (Java Server Pages)

Es un servidor que permite la creacion de un metodo dinamico independiente de la plataforma para poder crear aplicaciones basadas en la Web. JSP puede accesar a toda la familia API de Java.[4]

Ajax

Termino que permite describir un modo de utilizar al mismo tiempo varias tecnologias (HTML, XHTML, CSS, Java Script, DOM, XML, XSLT). Al combinar esdtas tecnologias se puede lograr una aplicacion web capaz de actualizarce continuamente sin tener que cargar la pagina.[5]

Investigación 1: Ciclo de vida y de desarrollo de los sistemas de información

Ciclo de vida de los sistemas de información

  • Investigacion preliminar.
  • Definicion de requerimientos
  • Analisis y diseño de software.
  • Implementacion y prueba de unidades.
  • Integracion y prueba del sistema.
  • Operacion y mantenimiento.

Ciclo de desarrollo de sistemas de información

Sistema que incluye a personas, maquinas y metodos que se utilizan para recopilar, procesar, transmitir datos. Un sistema de informacion agrupa la infraestructura, la organizacion, el personal y todos los componentes necesarios para la recopilacion, procesamiento, almacenamiento, transmision, visualizacion, diseminacion y organizacion de la informacion. Todos estos sistemas pasan por una serie de fases a lo largo de su vida, estas fases son las siguientes:

  • Planificación: Realizar trabajos antes de la elaboracion que serviran de guia para el proyecto.
  • Análisis: Se entiende que es lo que debe de realizar el proyecto, se llega a una comprension adecuada de los requerimientos del sistema.
  • Diseño: Estudiar otras alternativas de implementacion para nuestro sistema, ademas de definir la estructura que tendra. Realizacion de diagramas.
  • Implementación: Selecion de las herramientas adecuadas para el desarrollo del sistema, todo basandose en el diseño.
  • Pruebas: Sirven para detectar errores para despues corregirlos.
  • Instalación o despliegue: Planificar el entorno en el que se ejecutara el sistema, hardware y software.
  • Uso y mantenimiento: Eliminar defectos, si aparecen nuevas necesidades hay que satisfacerlas. Esta parte consume de 40 a 80 % de los recursos de una empresa desarrolladora de software.[11]
